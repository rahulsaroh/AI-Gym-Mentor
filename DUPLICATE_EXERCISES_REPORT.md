## Root Cause Analysis

The duplicate IDs (0088, 1371, etc.) are **numeric-only** while the yuhonas_* IDs follow the format `yuhonas_*`. This indicates:

1. **Multiple data sources were merged**: One source used numeric IDs (0001-3194), and another source used yuhonas_* IDs
2. **Both sources contained overlapping exercises**: The same exercises appear in both datasets
3. **De-duplication at build time was incomplete**: The Python script (`exercise_db_seeder.dart`) mentions deduplication but numeric-ID entries were kept alongside yuhonas_* entries where names differed only in case

## Recommended Actions

### Priority 1: Fix Exact Duplicates (6 groups)
These are true duplicates that should be **merged or removed**:
- Keep one entry per duplicate pair (preferably the yuhonas_* version)
- Remove the numeric-ID duplicate
- Ensure no relationships (workout history, etc.) are lost

### Priority 2: Normalize Case Variants (112 groups)
Standardize exercise names to **Title Case** format:
- "barbell curl" → "Barbell Curl"
- "3/4 sit-up" → "3/4 Sit-Up"
- Keep the yuhonas_* IDs (they appear to be the canonical versions)

### Priority 3: Database Cleanup
Implement a **uniqueness constraint** on `(name, category, equipment)` at the database level to prevent future duplicates.

### Priority 4: Build Process Enhancement
Update the seeder script (`lib/features/exercise_database/data/datasources/exercise_db_seeder.dart`) to:
1. Prefer yuhonas_* IDs over numeric IDs when both exist
2. Normalize case during ingestion
3. Log any duplicates found during seeding

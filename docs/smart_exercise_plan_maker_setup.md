# Smart Exercise Plan Maker - Implementation Plan

This document outlines the setup and implementation of the "Smart Exercise Plan Maker" feature in AI Gym Mentor.

## 1. Overview
The Smart Exercise Plan Maker will allow users to input their preferences and receive a tailored workout plan (as a `WorkoutTemplate` or `Mesocycle`) generated by the Gemini AI.

## 2. Requirements
- **Equipment Selection**: Multi-select (Dumbbells, Barbell, Kettlebell, Bodyweight, Full Gym).
- **Fitness Goal**: Dropdown (Muscle Building, Fat Loss/Cutting, Strength, Endurance).
- **Experience Level**: (Beginner, Intermediate, Advanced).
- **Frequency**: Days per week (1-7).
- **Workout Duration**: Minutes (15, 30, 45, 60+).

## 3. Data Models

### `WorkoutGenerationRequest`
```dart
class WorkoutGenerationRequest {
  final List<String> equipment;
  final String goal;
  final String experienceLevel;
  final int daysPerWeek;
  final int sessionDuration;
  final List<String>? targetBodyParts;

  WorkoutGenerationRequest({
    required this.equipment,
    required this.goal,
    required this.experienceLevel,
    required this.daysPerWeek,
    required this.sessionDuration,
    this.targetBodyParts,
  });
}
```

## 4. Backend Logic (AI Integration)

The `GeminiService` already has a base for generating workouts. We will expand it to handle goals and advanced parameters.

### Updated Prompt Strategy:
```text
Act as a world-class strength coach. Generate a {request.daysPerWeek}-day {request.goal} program for a {request.experienceLevel} trainee.
Available Equipment: {request.equipment}
Session Duration: {request.sessionDuration} minutes.

The plan must be returned as a valid JSON array of days, where each day contains a list of exercises with sets, reps, and RPE cues.
```

## 5. UI Architecture

### Steps for the Wizard:
1. **Equipment**: Visual grid of equipment icons.
2. **Goal & Experience**: Selection chips.
3. **Schedule**: Frequency and duration sliders.
4. **Generating**: A beautiful "AI is thinking" animation with fitness-related status text.
5. **Review & Save**: Display the generated plan before saving it to the database.

## 6. Implementation Steps

### Phase 1: Service Layer
- Update `lib/features/exercise_database/data/datasources/gemini_service.dart` to include a more robust `generateFullProgram` method.
- Add logic to map AI-suggested exercises to IDs in the local `Exercises` table.

### Phase 2: State Management
- Create `lib/features/programs/providers/smart_plan_provider.dart` using Riverpod.
- Handle loading states, error handling (especially for AI timeouts), and the final saving of the template.

### Phase 3: Presentation
- Create `lib/features/programs/smart_plan_maker_screen.dart`.
- Use a `PageController` for the multi-step form.
- Implement `Lottie` animations for the generation phase to provide a premium feel.

### Phase 4: Integration
- Add a "Smart AI Generator" card/button in `ProgramsScreen`.
- Register the new route in `lib/core/router/router.dart`.

## 7. Success Metrics
- Users can generate a valid, equipment-compatible plan in under 15 seconds.
- Plans generated include proper warm-ups and a logical exercise order (compounds first).
- 100% compatibility with the existing `WorkoutTemplates` schema.
